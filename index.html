Gradient Descent vs. BFGS Optimization

Visualizing convergence paths on 2D functions. Enter a function below using x and y.
Notice how Gradient Descent struggles with highly curved landscapes, while BFGS intelligently maps the curvature.

Understanding the Parameters:

  • f(x, y) The mathematical landscape (objective function). Think of it as physical terrain with hills and valleys. Both algorithms are trying to find the absolute lowest point on this map.
  • Start X & Y The exact coordinates where both algorithms are dropped off to begin their search. Changing the start point can drastically change the path they take.
  • GD Learning Rate The step size for Gradient Descent. If it's too small, progress is incredibly slow. If it's too large, it zig-zags wildly across the valley or completely bounces out (diverges).
